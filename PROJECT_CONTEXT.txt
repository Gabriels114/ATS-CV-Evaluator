==============================================================================
  QUÉ HACE EL PROYECTO
==============================================================================

Evalúa qué tan bien un CV encaja con una oferta de trabajo (ATS score 0-100).
Dos modos de extracción:
  - LOCAL  : 100% local, sin API, regex + taxonomía de skills + spaCy opcional
  - LLM    : Claude API (claude-sonnet-4-6) con tool-use, más preciso

7 dimensiones con pesos industria:
  Hard Skills     30%   coincidencia de skills técnicos
  Experience      25%   años, relevancia, recencia
  Education       15%   grado, campo, certificaciones
  Title Alignment 10%   título actual/pasado vs título del JD
  Achievements    10%   logros cuantificados
  Soft Skills      5%   liderazgo, comunicación, etc.
  Formatting       5%   legibilidad ATS (headers, bullets, espaciado)

Umbrales de interpretación:
  80+  → Strong Match   (pasa filtros ATS, probable entrevista)
  60-79 → Acceptable   (cubre requisitos clave)
  40-59 → At Risk       (faltan skills importantes)
  <40  → Low Match      (probable rechazo automático)

==============================================================================
  DECISIONES DE DISEÑO IMPORTANTES
==============================================================================

1. INMUTABILIDAD: todos los modelos son @dataclass(frozen=True, slots=True).
   Nunca mutar, siempre crear instancias nuevas.

2. SKILLS_DB precompilado: las regexes de matching se compilan una vez al
   importar el módulo. No recompilar en cada llamada (era O(n×m), ahora O(1)).

3. PLACEMENT BOOST: skills encontradas en sección summary o skills del CV
   reciben multiplicador 1.3x en el score de hard_skills.

4. RECENCY DECAY: experiencia reciente (< 2 años) → factor 1.0.
   Decae exponencialmente hasta ~0.3 para experiencias de > 10 años.

5. date.today() DENTRO de score(): no como constante de módulo. Así los tests
   no dependen de cuándo se importó el módulo.

6. WEB TEMP FILES: el endpoint /evaluate usa try/finally para garantizar
   limpieza del archivo temporal del CV aunque ocurra una excepción.

7. SEMANTIC MATCHER: lazy-load en primer uso, graceful fallback si
   sentence-transformers no está instalado (devuelve set vacío / 0.0).

8. CONTACT SUGGESTIONS: la función generate_suggestions() acepta
   contact: ContactInfo | None para sugerir agregar email/LinkedIn/teléfono.

==============================================================================
  BUGS CONOCIDOS / RESUELTOS (historial útil)
==============================================================================

✅ "developer" en MID synonyms causaba que "Junior Developer" → MID.
   Solución: "developer" eliminado de MID synonyms en title_alignment.py.

✅ Duplicate Education entries cuando múltiples líneas del mismo bloque
   coincidían con _RE_DEGREE. Solución: consumed_up_to high-water mark.

✅ _split_required_preferred sólo separaba en double newlines, no detectaba
   "Nice to have: X, Y" inline. Solución: scan línea a línea con regex.

✅ Stale module-level _TODAY = date.today() en experience.py. Estaba al importar.
   Solución: llamar date.today() dentro de score() y pasarla como parámetro.

✅ SyntaxWarning en JS template literals dentro de strings Python.
   Solución: usar createElement/textContent en lugar de template literals JS.

✅ "ats: command not found" aunque el venv esté activado.
   Causa: shebang del script apunta al venv viejo si se renombra.
   Solución: pip install -e ".[web]" (reinstala y regenera el script).
